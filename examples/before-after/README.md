# Before / after

Same night. Two shapes.

| File | What you get |
|------|----------------|
| [before.md](./before.md) | Status dump: lists, greens, undecided pile |
| [after.md](./after.md) | OpenBook essay: sixty-second, chapters, five pen-ready asks, receipts |

**How to compare:** read only the sixty-second version and What's next in `after.md`, then skim `before.md`. You can decide from the essay's front matter; the dump makes you re-enter the terminal.

Length is not the point. Trust and finishability are. The after example is long on purpose when the night earned it (~15+ pages rendered). Compare shape and trust, not "shorter is better."
